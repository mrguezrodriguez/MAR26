📐 Parte I — El algoritmo de asignación

La idea en una frase

Cada alumno tiene un vector de dos coordenadas [volar, comer]. Queremos asignarle la clase extraescolar del alumno de referencia al que esté más cerca en ese espacio 2D.

Analogía Imagina que cada alumno es un punto en un plano. Los 3 alumnos de referencia (Rodrigo, Gabriela, Mateo) son como «capitales». Cada alumno nuevo va a la capital más cercana. Eso es, conceptualmente, lo que hace el algoritmo k-NN.

¿Qué es np.linalg.norm?

La distancia euclidea entre dos puntos es una generalización del teorema de Pitágoras que funciona para cualquier número de dimensiones:

2D (volar + comer)
d = √ ( (v₂−v₁)² + (c₂−c₁)² )
nD — funciona igual para vectores de tamaño 5, 100, 1000…
d = √ ( (x₁₂−x₁₁)² + (x₂₂−x₂₁)² + … + (xₙ₂−xₙ₁)² )

np.linalg.norm(v1 - v2) suma los cuadrados de todas las diferencias y aplica la raíz, sin importar el tamaño del vector. El código no cambia nada si mañana los alumnos tienen 5 características en vez de 2. 

import numpy as np

# Vectores de tamaño 2 — distancia euclidea 2D
v1 = np.array([3, 2])
v2 = np.array([0, 0])
np.linalg.norm(v1 - v2)   # √(3²+2²) = √13 ≈ 3.60

# Vectores de tamaño 5 — exactamente el mismo código
v1 = np.array([3, 2, -1, 4, 0])
v2 = np.array([0, 0,  0, 0, 0])
np.linalg.norm(v1 - v2)   # √(3²+2²+1²+4²+0²) = √30 ≈ 5.47

Algoritmo antiguo vs. algoritmo con NumPy

La única misión del ejercicio es reemplazar el patrón manual de centinela por dos funciones de NumPy:

⚠️ distancia_2d no existe en Python En el notebook de la práctica aparece distancia_2d(vector, vec_ref, 1). Eso no es una función real — es pseudocódigo puesto ahí como marcador para indicarte exactamente qué línea tienes que sustituir por np.linalg.norm. Si intentas ejecutarla tal cual obtendrás un NameError.
❌ Antes — pseudocódigo del enunciado
# Centinela: valor imposiblemente alto
distancia_minima = 99999999
indice_min = -1

for i, ref in enumerate(alumnos_ref):
    # ← distancia_2d NO existe, es un marcador
    d = distancia_2d(vec, ref)
    if d < distancia_minima:
        distancia_minima = d
        indice_min = i

# El índice ganador está en indice_min
✅ Ahora — con NumPy
# Guardamos las 3 distancias en un array
dists = np.zeros(3)

for i, ref in enumerate(alumnos_ref):
    d = np.linalg.norm(vec - ref)
    dists[i] = d

# np.argmin encuentra el índice mínimo solo
indice_min = np.argmin(dists)  ← clave
¿Por qué 99999999 y -1? Son valores centinela: el 99999999 garantiza que la primera distancia real siempre sea menor, y el -1 indica «todavía no tenemos ganador». Con np.argmin ya no necesitamos esa trampa: NumPy recorre el array y nos da el índice directamente.

Las 3 funciones que necesitáis

np.zeros(n)
Crea un array de n ceros. Sirve para inicializar el vector de distancias.
np.zeros(3) → [0., 0., 0.]
np.linalg.norm(v)
Devuelve la longitud (módulo) del vector v. Distancia euclídea.
norm([3,4]) → 5.0
np.argmin(arr)
Devuelve el ÍNDICE del valor mínimo. No confundir con np.min().
argmin([8,2,5]) → 1

Código completo del bucle (solución)

alumnos_ref  = ["Rodrigo", "Gabriela", "Mateo"]
curso_asignado = ["CAI", "ER", "NMPC"]
matriz_zero = np.zeros((12, 3))   # 12 alumnos × 3 referencias

for i_vector in range(num_alumnos):

    alumno       = lista_alumnos[i_vector]
    vector_alumno = alumnos_aidone[i_vector]   # array NumPy, necesario para restar

    dists = np.zeros(3)                       # 1. inicializar

    for idx_ref, alumno_ref in enumerate(alumnos_ref):
        idx_en_lista = lista_alumnos.index(alumno_ref)
        vec_ref      = alumnos_aidone[idx_en_lista]

        distancia    = np.linalg.norm(vector_alumno - vec_ref)  # 2. calcular
        dists[idx_ref] = round(distancia, 2)

    matriz_zero[i_vector] = dists

    elegido = np.argmin(dists)                               # 3. mínimo
    print(f"{alumno} → {curso_asignado[elegido]}")
Importante alumnos_aidone debe ser un array de NumPy (no una lista de listas) para que vector_alumno - vec_ref funcione elemento a elemento. Convertidlo con np.array(alumnos_aidone) antes del bucle.

Los pasos del ejercicio

  1. Crear el tablero
    np.full((10, 10), " ") — array 10×10 relleno de espacios. Buena práctica: convertirlo en función con parámetro lado.
  2. Posicionar barcos
    Un barco es una lista de tuplas [(fila, col), ...]. Iterar sobre esas tuplas y asignar tablero[pieza] = "O".
  3. Disparar
    Comprobar el valor de la celda: si es "O" → "X" (tocado); si es " " → "-" (agua). Usar un if/elif o match/case.
  4. Barco aleatorio
    Elegir posición inicial con random.randint y orientación con random.choice(["N","S","E","O"]). Calcular las celdas del barco, verificar que no se salen del tablero ni chocan con otro barco. Repetir si falla.

Lógica del barco aleatorio (lo más difícil)

def crea_barco_aleatorio(tablero, eslora=4):
    max_f = tablero.shape[0] - 1
    max_c = tablero.shape[1] - 1

    while True:                                     # intentar hasta conseguirlo
        barco = []
        fila    = random.randint(0, max_f)
        columna = random.randint(0, max_c)
        orient  = random.choice(["N","S","E","O"])

        for _ in range(eslora):
            barco.append((fila, columna))
            if   orient == "N": fila -= 1
            elif orient == "S": fila += 1
            elif orient == "E": columna += 1
            elif orient == "O": columna -= 1

        # Verificar validez
        valido = True
        for (f, c) in barco:
            if not (0 <= f <= max_f and 0 <= c <= max_c):
                valido = False; break
            if tablero[(f,c)] == "O":
                valido = False; break

        if valido:
            for pieza in barco:
                tablero[pieza] = "O"
            return tablero

⚠️ Errores más comunes

  • 🔴
    Olvidar convertir alumnos_aidone a array NumPy
    Si alumnos_aidone sigue siendo una lista de listas, la resta vector_alumno - vec_ref falla porque Python no sabe restar listas elemento a elemento.
    ✅ Añadir alumnos_aidone = np.array(alumnos_aidone) antes del bucle.
  • 🔴
    Confundir np.argmin con np.min
    np.min([8,2,5]) devuelve 2 (el valor mínimo).
    np.argmin([8,2,5]) devuelve 1 (el índice del mínimo). Necesitamos el índice para acceder a curso_asignado[indice].
    ✅ Usar siempre np.argmin en este algoritmo.
  • 🔴
    No importar numpy antes del bucle principal
    Si la celda con import numpy as np no se ha ejecutado, todas las llamadas a np.* fallan con NameError.
    ✅ Ejecutar las celdas en orden de arriba a abajo (o usar «Run All»).
  • 🔴
    En el barco aleatorio, salirse del tablero sin validar
    Si la posición inicial es (9,0) con orientación Norte, las piezas llegan a fila -1, lo cual en NumPy no da error pero accede a la última fila, silenciosamente incorrecto.
    ✅ Validar explícitamente que 0 <= fila <= max_fila para cada pieza.

💬 Preguntas para reflexionar

Pregunta Pista / respuesta clave
¿Por qué el algoritmo original usa 99999999 y -1? Son centinelas. NumPy los elimina con np.argmin.
¿Qué diferencia hay entre np.min y np.argmin? Uno devuelve el valor, el otro el índice. Aquí necesitamos el índice.
¿Por qué no podemos restar listas de Python directamente? Las listas no tienen operaciones matemáticas definidas. Los arrays NumPy sí.
¿Qué pasa si el barco aleatorio se sale siempre del tablero? El bucle while True seguiría indefinidamente. Por eso conviene un límite de intentos.
¿Para qué sirve tablero.copy() antes de validar el barco? Para no modificar el tablero original si el barco resulta ser inválido.
