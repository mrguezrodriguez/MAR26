🥐 Parte I — El problema de las pastelerías

La idea en una frase

Tenemos una matriz de cantidades (3 personas × 4 productos) y una matriz de precios (4 productos × 2 tiendas). Multiplicarlas da directamente el coste de cada persona en cada tienda — sin un solo bucle.

La magia del producto matricial Lo que a mano es una suma de 4 multiplicaciones por cada par (persona, tienda) — 3×2 = 6 cálculos en total — NumPy lo resuelve en una sola línea: cantidad @ precios.

¿Por qué hay que trasponer los precios?

Para multiplicar dos matrices, las dimensiones internas deben coincidir: (A × B) requiere que las columnas de A = filas de B.

Cantidades (3 × 4) × Precios (4 × 2) = Coste (3 × 2)
Si no traspusiéramos, Precios sería (2 × 4) → incompatible
precios_completo = np.array([precios_manolo, precios_starbucks])
# Forma: (2, 4) — dos filas, cada fila es una tienda

precios_completo = precios_completo.T
# Forma: (4, 2) — cuatro filas (productos), dos columnas (tiendas)

matriz_coste = cantidad_necesaria.dot(precios_completo)
# (3,4) @ (4,2)  →  (3,2) ✓

Equivalencias: tres formas de hacer lo mismo

# Las tres líneas producen exactamente el mismo resultado
resultado = cantidad_necesaria @ precios_completo
resultado = np.dot(cantidad_necesaria, precios_completo)
resultado = cantidad_necesaria.dot(precios_completo)

🎬 Parte II — Similitud Coseno

¿Qué mide el coseno?

La similitud coseno mide el ángulo entre dos vectores, ignorando su longitud. Si dos vectores apuntan en la misma dirección, coseno = 1 (idénticos). Si son perpendiculares, coseno = 0 (sin relación). Si son opuestos, coseno = −1.

Fórmula de similitud coseno
cos(θ) = (A · B) / (‖A‖ × ‖B‖)
A · B = producto escalar  |  ‖A‖ = módulo (norma) de A

Menor ángulo → mayor similitud. Toy Story y Jumanji comparten "Aventura" → están más cerca.

Los vectores de películas

Cada película se codifica como un vector binario: 1 si tiene ese género, 0 si no.

El código paso a paso

for pelicula1, vector in peliculas.items():
    coseno_maximo = -1                # ← centinela: mínimo absoluto posible
    for pelicula2, vector2 in peliculas.items():
        if pelicula1 != pelicula2:      # no comparamos una película consigo misma
            coseno = np.dot(vector, vector2) / (
                np.linalg.norm(vector) * np.linalg.norm(vector2)
            )
            if coseno > coseno_maximo:  # buscamos el máximo
                pelicula_similar = pelicula2
                coseno_maximo    = coseno

    print(f"Para {pelicula1} la más parecida es {pelicula_similar}")

¿Por qué coseno_maximo = −1?

Es un valor centinela: un punto de partida tan bajo que cualquier resultado real lo supera.

¿Qué pasaría con 0?
coseno_maximo = 0
Si todas las películas comparadas tuvieran coseno negativo (sin géneros en común), ninguna superaría el 0 y el resultado quedaría sin actualizar. Bug silencioso.
Con −1 (mínimo absoluto)
coseno_maximo = −1
El coseno siempre está en [−1, 1]. Inicializando en −1 garantizamos que el primer valor real siempre gane y pelicula_similar siempre se actualice.

🌍 Similitud coseno en el mundo real

Este algoritmo de película es exactamente el mismo mecanismo que usan sistemas de producción reales. Cambia el dominio, no la matemática.

📝 NLP — Comparación de textos

Cada texto se convierte en un vector numérico (TF-IDF, embeddings de OpenAI, BERT…). Luego mides similitud coseno entre ellos. 

# "El perro corre" vs "El can corre"
# → similitud alta (~0.95) misma idea

# "El perro corre" vs "La bolsa sube"
# → similitud baja (~0.12) sin relación

Lo usan motores de búsqueda semántica, detección de plagio y clustering de noticias.

De hecho en IO Lab — cuando analizas reseñas de Google Maps, los embeddings de esas reseñas se comparan con similitud coseno para agrupar opiniones similares. Exactamente esto, a escala.

🎵 Sistemas de recomendación

Cada usuario o producto es un vector de preferencias (puntuaciones, clics, tiempo). Netflix, Spotify y Amazon usan esto constantemente. 

usuario_a = np.array([5, 4, 1, 0, 5])
usuario_b = np.array([4, 5, 0, 1, 4])

# Similitud alta → recomendar a B
# lo que le gustó a A

Las películas del enunciado son una versión mínima de esto: Toy Story y Jumanji son "usuarios" con gustos similares.

👁 Computer Vision

Una imagen se convierte en un vector de features (CNN, ViT…). La similitud coseno dice si dos imágenes representan la misma cara, el mismo objeto o la misma escena.

Los sistemas de verificación facial (Face ID, control de acceso) comparan el vector de tu cara con el almacenado. Si el coseno supera un umbral (~0.95), eres tú.

🚨 Detección de anomalías

El vector de comportamiento de un usuario (páginas visitadas, horarios, geolocalización) se compara con su comportamiento habitual.

Si la similitud coseno entre "hoy" y "historial" es muy baja → comportamiento inusual → posible fraude o intrusión. Lo usan bancos y equipos de ciberseguridad.

⚖️ ¿Por qué coseno y no distancia euclidiana?

La clave es que el coseno ignora la magnitud del vector — solo le importa la dirección. En NLP esto es fundamental:

❌ Distancia euclidiana — problema
# Documento largo: menciona "Madrid" 10 veces
doc_largo  = [10, 2, 8, ...]

# Documento corto: menciona "Madrid" 2 veces
doc_corto  = [2,  0, 2, ...]

# Hablan del mismo tema pero...
# distancia euclidiana = enorme → ¿sin relación?
✅ Similitud coseno — solución
# Mismos vectores, diferente magnitud
doc_largo  = [10, 2, 8, ...]
doc_corto  = [2,  0, 2, ...]

# Apuntan en la misma dirección →
# coseno ≈ 0.99 → casi idénticos ✓

Resumen: usa distancia euclidiana cuando la magnitud importa (¿cuánto?). Usa similitud coseno cuando solo importa la dirección (¿de qué trata?).

MétricaMideCuándo usarla
np.linalg.norm(a − b) Distancia en el espacio Coordenadas físicas, asignación de clases (práctica Numpy II)
dot(a,b) / (‖a‖·‖b‖) Ángulo entre vectores Textos, preferencias, features de imágenes (esta práctica)

🧰 Funciones NumPy de esta práctica

np.dot(a, b)
Producto escalar (vectores) o matricial (matrices). El núcleo del coseno y del coste total.
np.dot([1,0,1], [1,1,0]) → 1
np.linalg.norm(v)
Norma (módulo) de un vector: √(x²+y²+…). Necesaria para normalizar en similitud coseno.
np.linalg.norm([3,4]) → 5.0
array.T
Transpuesta de una matriz. Intercambia filas y columnas para poder hacer el producto.
np.array([[1,2],[3,4]]).T
np.argmin(arr)
Devuelve el índice del valor mínimo. Útil para encontrar la tienda más barata.
np.argmin([50, 49]) → 1
array.dot(b)
Método equivalente a np.dot. Se puede encadenar como método del array.
cantidad.dot(precios)
@ operador
Operador de producto matricial en Python. La forma más legible y moderna.
cantidad @ precios
