#1 API — The Movie Database (TMDb)

1.1 Identificar parámetros del endpoint /discover/movie ⏱ 5 min
Investiga la documentación de TMDb e identifica los parámetros necesarios para obtener las 20 películas americanas y las 20 españolas más populares de 2023.
Conceptos clave:
  • Endpoint Discover: /discover/movie permite filtrar películas por múltiples criterios (año, país de origen, popularidad, etc.). Es el más potente de la API.
  • Paginación: La API devuelve 20 resultados por página por defecto. Si no se especifica page se obtiene la primera página (20 resultados).
Parámetros necesarios:

with_origin_country — código ISO del país (US o ES). Solo admite un valor → hay que hacer dos consultas separadas.
year o primary_release_year — año de estreno (2023).
sort_by=popularity.desc — ordena por popularidad descendente.
language=es-ES — (extra) devuelve campos traducidos como overview y title.
1.2 Llamada USA — parámetros en la URL (querystring) ⏱ 7 min
Construye url_usa con todos los parámetros en la propia URL mediante la sintaxis ?param=val&param2=val2 y lanza la petición a respuesta_usa.
Conceptos clave:
  • Querystring en URL: método directo de enviar parámetros GET. Se concatena todo como un string. Es transparente y fácil de depurar pegando la URL en el navegador.
  • El método correcto es requests.get() — las APIs REST de lectura usan siempre GET.
  • El país de Estados Unidos en ISO 3166-1 es US (no USA).
base_url = "https://api.themoviedb.org/3"
endpoint_discover = "/discover/movie"

querystring = (
    f"?api_key={api_key}"
    "&with_origin_country=US"
    "&year=2023"
    "&sort_by=popularity.desc"
    "&language=es-ES"   # EXTRA: resultados en español
)

url_usa = base_url + endpoint_discover + querystring

respuesta_usa = requests.get(url_usa)
Error frecuente: Usar USA en lugar de US como valor de with_origin_country. La API no devolvería resultados.
Extra: Añadir &language=es-ES hace que campos como overview y title se devuelvan en español cuando exista traducción.
1.3 Código de respuesta, JSON y extracción de pelis_usa ⏱ 5 min
Muestra el código de respuesta, vuelca el JSON en datos_usa, examina sus campos y asigna el campo con los datos a pelis_usa.
Conceptos clave:

  • El JSON de TMDb tiene siempre 4 campos de primer nivel: page, results, total_pages, total_results. Los datos están en results.
  • .json() de requests parsea automáticamente la respuesta a dict/lista Python.

1.8 Lanzar petición España con argumento params ⏱ 2 min
Lanza la petición usando el argumento params de requests (no la URL) y muestra status code y reason.
Conceptos clave: Cuando se pasa un diccionario al argumento params, requests.get() lo convierte automáticamente en querystring y lo añade a la URL antes de enviar la petición. El resultado es equivalente al método anterior, pero más robusto (gestiona encoding de caracteres especiales). 
respuesta_spain = requests.get(url_spain, params=querystring)

# → 200 OK
Diferencia clave: params → querystring (GET). data/json → body de la petición (POST/PUT). Las APIs REST de lectura usan siempre params.

#2 Web Scraping — BoxOfficeMojo

2.1 EXTRA Scrapear la tabla de ingresos de BoxOfficeMojo con BeautifulSoup ⏱ 20 min
Acceder a https://www.boxofficemojo.com/year/2023/?area=ES y extraer la tabla de datos en un DataFrame.
Proceso de inspección del HTML:
  1. Abrir DevTools → <body> → <main> → <div class="a-section mojo-body">
  2. Buscar <div id="table"> → <div class="imdb-scroll-table-inner">
  3. Columnas: primera <table> → tags <th> → texto de cada uno.
  4. Filas de datos: segunda <table> → <tbody> → tags <tr> → tags <td> con los valores.
Truco del índice de columna: Al iterar las <td> de cada <tr> con enumerate, el índice indica a qué columna pertenece cada valor → se puede poblar el diccionario directamente sin parsear atributos. 
url_mojo = "https://www.boxofficemojo.com/year/2023/?area=ES"

# 1. Petición y parseo
mojo = requests.get(url_mojo)
print(mojo.status_code, mojo.reason)  # → 200 OK
sopa = bs(mojo.text, "lxml")

# 2. Extraer nombres de columnas (tags <th> del primer <table>)
columnas_mojo = []
tabla = sopa.find("table")
for th in tabla.find_all("th"):
    columnas_mojo.append(th.text.strip())

print(columnas_mojo)
# → ['Rank','Release','Genre','Budget','Running Time','Gross',
#    'Theaters','Total Gross','Release Date','Distributor','Estimated']

# 3. Inicializar diccionario de listas (una lista por columna)
diccionario_data = {col: [] for col in columnas_mojo}

# 4. Extraer filas de datos (tags <tr> → <td>)
for fila in sopa.find_all("tr"):
    for idx, celda in enumerate(fila.find_all("td")):
        col = columnas_mojo[idx]
        diccionario_data[col].append(celda.text.strip())

# 5. Crear DataFrame
df_mojo = pd.DataFrame(diccionario_data)

# 6. Limpiar nombres de columnas y establecer índice
df_mojo.columns = [col.lower().replace(" ", "_") for col in df_mojo.columns]
df_mojo = df_mojo.set_index("rank")

2.3 Merge: pegar recaudaciones a las 40 películas (LEFT JOIN) ⏱ 5 min
Pegar las recaudaciones de df_mojo a df_base_movies, manteniendo siempre las 40 películas. Antes responder: ¿qué método? ¿qué tipo de join? ¿qué campos son los de cruce?
Método: merge() — necesitamos cruzar por una columna, no apilar filas.

Tipo de join: LEFT JOIN — queremos mantener las 40 películas de df_base_movies aunque no tengan recaudación en Mojo. Las películas sin match quedan con NaN en las columnas de Mojo.

Columnas de cruce: original_title en df_base_movies ↔ release en df_mojo (son columnas con nombres distintos → usar left_on / right_on).
¿Por qué no podemos cruzar por id? Porque df_mojo viene de un scraping HTML y no contiene el ID interno de TMDb. El único campo en común es el título de la película. 
df_total = df_base_movies.merge(
    df_mojo,
    left_on="original_title",   # columna de cruce en df_base_movies
    right_on="release",          # columna de cruce en df_mojo
    how="left"                  # mantener todas las películas de la izquierda
)
Advertencia importante: El cruce por título es impreciso: si el título en TMDb y en BoxOfficeMojo difiere ligeramente (tildes, traducción, etc.) no habrá match. En producción se usaría el IMDB ID que está disponible en ambas fuentes.

2.5 Películas españolas con información de ingresos ⏱ 5 min
Muestra las películas españolas para las que sí hay información de ingresos. Cuidado: tiene "truco".
El truco: df_total no tiene ninguna columna que indique si la película es española o americana (ese dato se perdió al hacer el concat). Hay que reconstruirlo sabiendo que en df_base_movies las primeras 20 filas son USA y las 20 siguientes son ES.
Dos formas de detectar si hay datos de ingresos:
  • df_total.gross.isna() == False → filas donde la columna gross no es NaN (tiene recaudación).
  • El LEFT JOIN deja NaN en todas las columnas de Mojo cuando no hay coincidencia, así que cualquiera de esas columnas sirve como indicador.
# Paso 1: marcar el país de origen (USA primeras 20, ES últimas 20)
df_total["country"] = "USA"
df_total.iloc[20:, -1] = "ES"

# Paso 2: filtrar españolas con ingresos (gross no nulo)
df_total[(df_total.country == "ES") & (df_total.gross.isna() == False)]

2.6 INNER JOIN + correlación visual de rankings ⏱ 8 min
Crear un índice de popularidad en df_base_movies, hacer un INNER JOIN con df_mojo para quedarse solo con las películas que tienen recaudación, y comparar visualmente ambos rankings.
Conceptos clave:
  • Índice de popularidad: Las películas ya vienen ordenadas por popularidad dentro de cada grupo (USA 1-20, ES 1-20), por lo que basta con crear una lista [1..20, 1..20].
  • INNER JOIN: Solo devuelve las filas con coincidencia en ambas tablas → solo las películas que aparecen tanto en TMDb como en BoxOfficeMojo.
  • Antes del INNER hay que hacer un reset_index en df_mojo para no perder el campo rank (que era el índice).
# 1. Añadir índice de popularidad (1-20 por cada grupo)
df_base_movies["rank_pop"] = list(range(1, 21)) + list(range(1, 21))

# 2. Sacar rank de df_mojo del índice para no perderlo en el merge
df_mojo.reset_index(names="rank_ingresos", inplace=True)

# 3. INNER JOIN — solo películas con recaudación
df_total_inner = df_base_movies.merge(
    df_mojo,
    left_on="original_title",
    right_on="release"
    # how="inner" es el valor por defecto
)

# 4. Comparar rankings visualmente
df_total_inner.sort_values("rank_pop", inplace=True, ascending=False)
df_total_inner[["release", "rank_pop", "rank_ingresos"]]
