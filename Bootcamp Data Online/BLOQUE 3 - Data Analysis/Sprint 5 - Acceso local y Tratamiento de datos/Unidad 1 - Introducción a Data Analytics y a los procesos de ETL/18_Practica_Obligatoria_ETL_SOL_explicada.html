Calcular la cardinalidad de cada columna
Obtener la cardinalidad (% de valores únicos sobre total) y guardarla en un diccionario.
¿Qué es la cardinalidad? El porcentaje de valores únicos de una columna respecto al total de filas. Una cardinalidad del 100% significa que cada fila tiene un valor distinto → posible índice. Una cardinalidad muy baja indica una columna categórica con pocos valores distintos.
cardinalidad = (nº valores únicos / nº total de filas) × 100
# Versión explícita (con bucle for)
diccionario_cardinalidad = {}
for col in df.columns:
    cardinalidad = round(df[col].nunique() / len(df) * 100, 2)
    diccionario_cardinalidad[col] = cardinalidad

# Versión corta con Series (equivalente, más pythónica)
serie_cardinalidad = (df.nunique() / len(df) * 100).round(2)
diccionario_cardinalidad = dict(serie_cardinalidad)

#2 · Limpieza y transformación (I)

Strip · Cardinalidad baja · Simplificar tipos · Corregir rating · Convertir fechas

1
Eliminar espacios en blanco con strip()
Quitar espacios antes/después de cada valor string en todas las columnas.
Los espacios invisibles son un problema clásico en datos reales: "Ginebra" y " Ginebra" son distintos para Pandas aunque visualmente parezcan iguales. Se aplica str.strip() solo a columnas object (no se puede aplicar a numéricas). 
for col in df.columns:
    if df[col].dtype == "O":   # solo columnas de tipo object (strings)
        df[col] = df[col].str.strip()
¿Por qué comprobar el tipo antes de aplicar strip? Si se aplica .str.strip() a una columna entera como release_year (int64), Pandas lanzará un error porque los enteros no tienen métodos de cadena.

3a · rating: buscar el valor real para cada título
Son solo 7 filas (0.08%). La mejor estrategia es buscar el dato real en internet o en una IA y hardcodearlo en un diccionario. Es laborioso pero preciso.

¿Por qué no usar la moda para rating? TV-MA es la moda, pero no todos los títulos son TV-MA. Con tan pocos nulos, buscar el valor real es factible y mucho más preciso.
3b · date_added: imputar con la moda + crear flag
Solo 10 filas sin fecha. Es imposible saber cuándo se añadieron exactamente, así que usamos la moda (fecha más frecuente). Además creamos una columna flag para registrar que ese valor fue imputado. 
moda_added_date = df.date_added.mode()

4
BONUS: campo "Posible_Secuela" con expresiones regulares bonus
Crear un campo booleano que detecte si una película podría ser una secuela (el título de la película original también existe en el dataset).
La solución evoluciona en 3 versiones progresivas, cada una más precisa. Es un ejemplo perfecto de cómo se refinan las expresiones regulares en la práctica.
❌ Versión 1 — demasiado simple 
def es_posible_secuela(titulo):
    # Solo detecta títulos que SON un número: "2012", "678"...
    match = re.match("[0-9]+$", titulo)
    return match is not None
Problema: detecta películas como "1922" o "3022" que son títulos que solo son números, no secuelas.
⚠️ Versión 2 — mejor, pero imprecisa 
def es_posible_secuela(titulo):
    # Detecta: cualquier texto + espacio + número al final
    # "Jaws 2" ✓  "Shrek 2" ✓  "Rocky IV" ✗ (letras, no número)
    match = re.match(".* [0-9]+$", titulo)
    return match is not None
Detecta "Jaws 2", "Jaws 3"... pero también "Ocean's 11" (que no es secuela de nada).
✅ Versión 3 — la mejor: verifica que la película original existe 
def es_posible_secuela(titulo):
    # Grupo de captura: extrae el texto antes del número final
    match = re.match("(.*) [0-9]+$", titulo)
    if match is not None:
        # Extraemos el título base: "Jaws 2" → "Jaws"
        titulo_base = match.group(1).strip()
        # Comprobamos si "Jaws" existe en el dataset
        return len(df_movies[df_movies.title == titulo_base]) > 0
    return False

df_movies["Posible_Secuela"] = df_movies.title.apply(es_posible_secuela)

¿Cómo funciona el grupo de captura (.*)? El patrón "(.*) [0-9]+$" tiene dos partes:
  • (.*) — grupo 1: captura todo el texto hasta el último espacio
  • [0-9]+$ — espacio + uno o más dígitos al final
Con match.group(1) recuperamos el texto del grupo 1 (el título base).
💡 ¿Podría mejorarse aún más? Sí. Esta versión no detecta "Rocky IV" (número romano) ni "El Padrino II". Para casos más complejos habría que ampliar el patrón regex o usar un diccionario de números romanos. Pero para el alcance de la práctica, la versión 3 es excelente.
